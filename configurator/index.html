Self-host guide

        
Field guide

        
From a bare machine to raiding with friends. The happy path is five steps — expand any step for the full walkthrough.

      

      
    

        
  1. 
          
    Get a host + Docker
    
          
    You need a Linux machine with Docker and the Compose plugin. Already have one? Skip to step 2.
    
          
    
            Set up a host & Docker
            
    
              
    A free host
    
              
    No server lying around? Oracle Cloud's Always-Free tier is plenty — up to 4 Arm cores, 24 GB RAM, 200 GB disk. It's Arm, which the server image supports natively. But the headless client is x86-only — it won't run on an Arm host, so pick an x86 box if you want headless.
    
              
    Connect to it
    
              
    Easiest from Windows: VSCode with the Remote Explorer extension — edit files and run a terminal in one window over SSH. (PuTTY + WinSCP work too.) Keep your SSH key safe — lose it and you're locked out of the server.
    
              
    Install Docker
    
              
    Follow Docker's official guide (it stays current), or the short version on Ubuntu/Debian:
    
              
    curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER   # add yourself to the docker group
newgrp docker                   # apply it now — docker then runs without sudo
docker compose version          # verify (Compose plugin ships with it)
    
            
    
          
    
        
    2. 

        
  2. 
          
    Configure your server
    
          
    Set your options in the configurator above — SPT version, Fika, ports, and the QoL tab (ModSync + Quartermaster) — then hit Extract bundle (.zip).
    
          
    
            Quality-of-life: ModSync & Quartermaster
            
    
              
    ModSync — keep clients in sync
    
              
    Turn on Install ModSync and the server hosts the client-side mods: each player installs the ModSync plugin once, then auto-syncs everything else from the server. Without it, every player installs every mod by hand.
    
              
    Quartermaster (quma) — the mod manager
    
              
    Turn on Install Quartermaster for an advanced web UI that manages your server-side mods for you — search SPT Forge and install / update / remove mods without touching files, then restart the server from the browser. It serves admins and players: admins manage mods and the server; players get a page to grab the mods they need to join and can request more.
    
              
    Set an admin password (min 8 chars) — the bundle won't build without one while quma is on. After deploying, reach it at https://your-server-ip:9190 (self-signed cert — accept the browser warning) and sign in as admin, or put it behind your reverse proxy for a clean HTTPS domain.
    
              
    quma talks to the Docker socket to restart the server after changes, so the compose mounts /var/run/docker.sock and the same data directory as the server. Mods it installs land in that data dir like any other.
    
            
    
          
    
        
    3. 

        
  3. 
          
    Deploy it
    
          
    Copy the bundle to your host, unzip, and bring it up with docker compose up -d --wait.
    
          
    
            Deploy & manage commands
            
    
              
    unzip spt-fika-bundle-*.zip && cd spt-fika-bundle-*
docker compose up -d --wait     # --wait blocks until the server is healthy
docker compose logs -f          # watch Fika / ModSync install + boot
    
              
    Use --wait. The headless client, web app and Quartermaster are gated on the server's healthcheck (depends_on: service_healthy), so they only start once the server is up. Without --wait they can be left not started.
    
              
    No manual folder setup needed — Docker creates your data directory on first run, and the container seeds the SPT files into it and sets ownership automatically.
    
              
    Day to day:
    
              
    docker compose restart                                # restart
docker compose down                                   # stop (your data dir persists)
docker compose down && docker compose up -d --wait     # apply config changes
docker compose pull && docker compose up -d --wait     # update to a newer image tag
    
              
    Your data directory (set in GENERAL) holds profiles, mods and configs — back it up before changing SPT versions.
    
            
    
          
    
        
    4. 

        
  4. 
          
    Connect your client
    
          
    Install SPT + the matching Fika plugin on your PC, then point the SPT launcher at https://your-server-ip:6969.
    
          
    
            Client downloads (match your server version)
            
    
              
    Your client's SPT and Fika versions must match what you set on the server.
    
              
      
                
    • SPT 4.0: SPT + Fika plugin (4.0 release).
      • 
                
    • SPT 3.11: the 3.11 SPT release + matching Fika plugin. Note: the 3.11 server image is still in testing — 4.0 is the verified path.
      • 
              
    
              
    Using ModSync? Each client installs the ModSync plugin once, then auto-syncs the rest of the mods from the server.
    
            
    
          
    
        
    5. 

        
  5. 
          
    Bring in friends
    
          
    Open your game port so others can reach the server over the internet.
    
          
    
            Networking
            
    
              
    Allow/forward your game port (default 6969) on the host firewall and your router. On a cloud VPS, also open it in the provider's security rules (e.g. Oracle's ingress rules).
    
              
    Fika links players peer-to-peer for raids, which can need extra port-forwarding or UPnP on the host player's side — see Fika's documentation for the networking specifics.
    
              
    Reverse proxy (optional)
    
              
    To reach the web app over HTTPS at a domain instead of :8080 over http, put it behind a reverse proxy. Caddy is the easiest — automatic HTTPS, one block per service. Run it on the same Docker network as the stack so it reaches services by name, and drop the web app's ports: (no longer needed):
    
              
    webapp.your-domain.com {
    reverse_proxy spt-fika-webapp:5000
}

Briefing

        
Questions

      

      

        
Do I need to own Escape from Tarkov?
No game files are bundled or required to run the server. SPT is a single-player/co-op server stack — see the SPT and Fika projects for client setup.

        
Where do my profiles and mods live?
In the data directory you set (default ./server-data), bind-mounted into the container. They persist across restarts and updates.

        
How do I update SPT or Fika?
On 4.0: for SPT, pull a new image tag and docker compose up -d; for Fika, set AUTO_UPDATE_FIKA and bump the version — it reinstalls in place, keeping your config. 3.11 is frozen — SPT, Fika and ModSync are pinned and there are no updates.

        
Can friends join over the internet?
Yes — keep "Listen on all networks" on and forward your game port. Each player still installs the SPT launcher and Fika client.

        
x86 or ARM?
The image is multi-arch, so the server runs on either. The headless client is x86-only, so the configurator hides it when you pick ARM.

        
What does the headless client do?
It's an optional Fika client that runs unmanned to host raids, so players can join even when nobody's hosting. It's x86-only and off by default. It runs a real SPT client under the hood, so you mount your SPT + EFT headless client files into its data directory and set a headless profile ID (the server generates one on first boot, when Headless profiles ≥ 1). See Fika's headless guide.

        
How do I add other mods?
Enable Quartermaster in the QoL tab — its web UI searches SPT Forge and installs, updates and removes server-side mods for you, no files to touch. To sync those mods to players automatically, also enable ModSync. See the Configure your server step in the field guide for the walkthrough.

        
Does anything I enter get uploaded?
No. This is a static page with no backend — your options never leave the browser, and the bundle is built locally. You can even run it offline.

        
How are my profiles backed up?
SPT 4.0 backs up profiles itself. For extra safety, copy your data directory before changing SPT versions — it holds your profiles, mods and configs.

        
What is the Fika Web App?
An optional browser admin UI (toggle it on in the OPS tab) — accounts, item sending, profile up/download, headless control. After deploying, open http://your-server-ip:8080 and sign in with admin / Admin123! — change that password immediately. It needs an API key generated by the Fika server: set WEBAPP_API_KEY in your .env after first boot. It's HTTP-only unless you put it behind a reverse proxy. Full setup on the Fika wiki.

        
What is Quartermaster?
An advanced server web UI (toggle Install Quartermaster in the QoL tab) for admins and players. Admins search SPT Forge and install / update / remove server-side mods without touching files by hand, and restart the server from the browser — it mounts /var/run/docker.sock and the same data directory as the server. Players get a page to grab the mods they need to join and can request more. Set an admin password (min 8 chars) — written to .env as QUMA_ADMIN_PASSWORD and applied on first boot. It serves HTTPS on port 9190 (self-signed) — reach it directly at https://your-server-ip:9190 or behind your reverse proxy.
